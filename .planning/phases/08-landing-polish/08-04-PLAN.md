<task type="auto">
  <name>Task 2: Add accessibility improvements</name>
  <files>app/(marketing)/page.tsx, app/(marketing)/pricing/page.tsx, components/marketing/Header.tsx</files>
  <action>
Improve accessibility across marketing pages:

1. Add proper semantic HTML:
   - Use `<header>`, `<main>`, `<footer>`, `<nav>`, `<section>`
   - Use heading hierarchy (h1 → h2 → h3)
   - Add `<article>` for feature cards

2. Add ARIA labels:
   - Navigation: `aria-label="Main navigation"`
   - Mobile menu button: `aria-expanded`, `aria-controls`
   - CTA buttons: descriptive text (not just "Click here")

3. Ensure keyboard navigation:
   - All interactive elements focusable
   - Visible focus indicators (outline)
   - Skip link to main content

4. Color contrast:
   - Verify text meets WCAG AA (4.5:1 ratio)
   - Current blue (#3B82F6) on white is fine
   - Check secondary text colors

5. Add alt text to any images (OG image placeholder, etc.)

6. Ensure form labels on signup page are properly associated.
  </action>
  <verify>Lighthouse accessibility score > 90, keyboard navigation works</verify>
  <done>Semantic HTML, ARIA labels, keyboard nav, and contrast verified</done>
</task>
